# F1 Strategy Simulator (2026 season)

A per-driver Formula 1 pit-strategy tool for the 2026 season. Pick any round, and for each driver it
works out the pit-and-tyre strategy that gives that driver their best finish against a realistic
field, then reports the expected finish, win probability, and (for races already run) how the call
compares to what the team actually did.

It has two parts, kept separate on purpose:

1. The web app. Zero dependencies, plain ES modules and HTML5 canvas, no npm, no bundler, no build
   step. It only reads small JSON files from `data/`.
2. The calibration pipeline (`calibrate/`). Offline Python (FastF1) plus Node scripts that fetch real
   2026 timing and telemetry and write those JSON files. You run it on a networked machine; the app
   itself never runs Python.

## What it does and doesn't model

This is a strategy model, not a betting tool. Safety cars are simulated (a per-lap chance of a
full-course caution). It does not model retirements or DNFs, red flags, driver error, or mid-race
weather changes. Per-car pace comes from prior rounds only, so early-season predictions are the
weakest. Win percentages reflect the model's own assumptions, not real-world odds.

## Pages

- `index.html`, the race-day predictions home page. Per-driver strategy calls for every 2026 round,
  team-coloured, with a circuit map and, for completed rounds, predicted vs actual accuracy.
- `simulator.html`, the interactive simulator. Pick a circuit from the dropdown, set the grid, pick a
  driver and weather, and a genetic algorithm searches for the best strategy, then replays a synced
  race trace and track map with a live running-order tower.
- `references.html`, every external source used to calibrate the model.

## Run the app locally

The app uses ES modules and `fetch`, so it needs a static server (browsers block those over
`file://`). From the project root:

```bash
python3 -m http.server 8000
```

Then open http://localhost:8000/. Any static server works (`npx serve`, VS Code Live Server, and so
on). No build step. The console should log `[predictions] boot OK`.

## Reproduce the data end to end

Everything in `data/` is generated by the pipeline below. The only steps that need internet are the
Python fetch and extract scripts (they download from FastF1). The Node scripts are pure and run
offline against the JSON those produce.

### 0. One-time setup

```bash
conda env create -f calibrate/environment.yml   # creates env "f1-calibrate" (fastf1, pandas, numpy, scipy)
conda activate f1-calibrate
```

Node (v18+) runs the report and prediction scripts and needs no install.

### 1. Fetch real pace, strategies, and circuit outlines (Python, needs internet)

Run from the project root. The first run is slow (FastF1 caches each session into `calibrate/cache/`,
which is git-ignored); later runs are fast.

```bash
python calibrate/check_data.py                 # which 2026 sessions are available, into data/availability.json
python calibrate/fetch_pace.py --all           # per-car clean-air pace + corner/straight split, into data/pace/
python calibrate/fetch_strategies.py --all     # each driver's real stints + pit laps, into data/strategies/
python calibrate/extract_circuits.py --all     # track outlines + per-track constants, into data/tracks/ and data/races.json
python calibrate/extract_race_notes.py         # honest per-race "where the model couldn't help" notes (safety cars, DNFs), into data/race_notes.json
```

On a sprint weekend you can also fetch that weekend's Sprint pace (not a leak, it happens before the
race): `python calibrate/fetch_pace.py --rounds 9 --sprint` writes `data/pace/2026_R9_sprint.json`.

### 2. Generate predictions and reports (Node, offline)

```bash
node calibrate/generate_predictions.mjs               # writes data/predictions.json (drives index.html)
node calibrate/recommend.mjs --round 9 --weather DRY  # pre-race call, writes reports/2026_R9_prerace.md
node calibrate/review.mjs   --round 8                 # post-race review, writes reports/2026_R8_review.md
```

### 3. Validate (Node, offline)

```bash
node calibrate/backtest_all.mjs                # per-round recommended vs actual, writes reports/backtest_all_rounds.md
node calibrate/backtest_accuracy.mjs           # model accuracy vs reality (Spearman rho, winner/podium rate)
```

The leakage rule is automatic: predicting round N only ever uses pace from rounds before N. Round 1
has no prior 2026 data, so it falls back to flat pace and is the weakest prediction by design.

## How the model works (short version)

- Pace (`src/pace.js`): aggregate prior rounds' FastF1 clean-air fuel-corrected pace into a per-car
  delta, shrunk toward the field mean when data is thin, and tilted by how corner-heavy the target
  track is.
- Compounds (`src/compounds.js` and the `compound_model` in `data/constants.json`): each track's real
  Pirelli C1-C5 allocation (`data/races.json`) is mapped to soft/medium/hard, so a "soft" at
  Silverstone (C3) is durable and a "soft" at Austria (C5) degrades fast.
- Optimiser (`src/dp.js`): for each stop count from 1 to 4, an exact dynamic program finds the best
  stop laps for every tyre-legal compound sequence on that driver's own pace, then each candidate is
  Monte-Carlo refined against the field (`evaluateStrategy` in `src/ga.js`), the exact pit laps are
  tuned per driver, and the candidates are ranked by expected finish. The interactive simulator uses
  the genetic algorithm in `src/ga.js` instead; both optimise one driver at a time.
- Race engine (`src/engine.js`): a deterministic seeded per-track race with tyre degradation, fuel
  burn, pit stops, dirty air, a saturating overtaking model, the team double-stack rule, and a
  probabilistic safety car. Running it many times over different noise and safety-car seeds produces
  the finishing distribution.

## Project layout

```
index.html            race-day predictions (home)
simulator.html        interactive GA simulator (circuit picker)
references.html       external sources
/src
  engine.js           deterministic race engine
  field.js            reactive rival-field policies
  ga.js               genetic algorithm + Monte-Carlo evaluator
  dp.js               exact dynamic-programming optimiser (per stop-count)
  pace.js             aggregate FastF1 pace into per-car pace (leakage-safe)
  compounds.js        real Pirelli C1-C5 into track-accurate pace/deg
  predictions.js      home page controller
  main.js, ui.js      simulator bootstrap + controls
  render-trace.js, render-map.js   canvas views
/data
  constants.json      tyre / overtake / safety-car / fuel model
  races.json          per-round meta (laps, base lap, pit loss, compounds, corner share)
  grid.json           driver/team/colour metadata + default grid
  predictions.json    generated dataset behind index.html
  /pace /strategies /tracks   per-round JSON from the pipeline
/calibrate            offline FastF1 (Python) + Node report/backtest scripts
  environment.yml     conda env
```

## Requirements

- App: any static file server and a modern browser (ES modules). Nothing else.
- Pipeline: conda (or miniforge) for the `f1-calibrate` env, and Node 18+ for the report scripts.
